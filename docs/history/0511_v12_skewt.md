# 0511 v12_skewt — SDB Skew-t mixture extension of v11tm

## 1. Motivation

v11tm 의 elliptical Student-t mixture 가 MIDUS 의 비대칭 cluster shape (inflammation
오른쪽 skew, cognition 의 long-tail) 을 잡지 못하는 한계를 model-level 에서 해결하기 위해
SDB (Sahu-Dey-Branco 2003) skew-t 로 확장.  Density 적합도가 아니라 _partition
inference + LSIRM biplot geometry_ 가 inferential target 이라는 점에서 "component =
cluster" 패러다임을 유지하면서 cluster shape family 만 비대칭까지 허용.

## 2. 이번 세션의 작업 흐름

### 2.1 v11tm marginal_b 알고리즘 검증

- 결론: 수학적으로 OK, sweep order 도 valid.

### 2.2 v11tm marginal_b 캐시 최적화 — **완료**

leave-one-out NIW 예측의 매 b-MH 평가비용을 O(K* × P) → O(K* × d²) 로 축소.

- 신규 헬퍼 함수 3개 (`data/my_LSIRM_FMC_v11tm.cpp`):
  - `build_cluster_stats_cache`: 한 sweep 시작에 cluster 별 충분통계
    (W_l, sum_wz_l, sum_wzz_l, n_l) 구축
  - `niw_posterior_loo_from_cache`: cluster l 이 c_q 와 같으면 q 의 기여분만 빼서
    LOO posterior O(d²)
  - `log_marginal_b_prior_cached`: cache + LOO 로 log-sum-exp 평가
- b sweep 시작 시 1회 재구축, 매 accept 시 c_q cluster 만 증분 업데이트.

**Benchmark** (MIDUS scale n=221, P=68, K*=6, 1000 iter):
- marginal_b=TRUE (cached): 23.0s
- marginal_b=FALSE (conditional): 22.5s
- Overhead: **1.02x** (사실상 무료).  최적화 전엔 ~5-10x 격차 예상.

**부수 발견 (별개 버그)**: `n_save = (n_iter - burnin) / thin` 이
`(n_iter - burnin) % thin != 0` 일 때 OOB.  MIDUS driver (30000/10000/5) 는 정확히
나눠떨어져 production 영향 없음.  미고침 (스코프 외).

### 2.3 Mixture flexibility 확장 방향 논의

대안 검토:
1. Per-cluster $\nu_l$ — 낮은 비용, MIDUS 의 cluster heterogeneity 캡쳐
2. Pitman-Yor on $\rho$ — power-law cluster size, e0 sweep 양극단 완화
3. Skew-t (SDB) — **선택됨**. v11tm 의 nested superset, residual augmentation 으로
   캐시 구조 살릴 수 있음.
4. NIW κ_0, ν_0 hierarchical hyperprior
5. CSN (multi-direction skew), GH (mean-variance mixture) — over-flexible 로 보류

**선택 근거**: 시각적 효과 + nested-model 비교 가능성 + v11tm 인프라 재사용.

### 2.4 v12_skewt 구현 — **완료**

3개 파일:

| 파일 | 역할 |
|---|---|
| `data/my_LSIRM_FMC_v12_skewt.cpp` (2125 줄) | C++ MCMC kernel |
| `data/my_LSIRM_FMC_cpp_v12_skewt.R` (446 줄) | R wrapper + Procrustes (Δ co-rotation 포함) |
| `data/MIDUS_5layered_result_v12_skewt.R` (1444 줄) | MIDUS driver |

**모델 변경**:
$$w_q \sim \mathrm{Gamma}(\nu_t/2, \nu_t/2),\quad U_q \mid w_q \sim N_+(0, 1/w_q),\quad
b_q \mid c=l, w_q, U_q \sim N_d(\mu_l + \Delta_l U_q, \Sigma_l/w_q)$$

- $\Delta_l \in \mathbb{R}^d$: 클러스터별 skew 방향.  Prior $\Delta_l \sim N_d(0, \tau_\Delta^2 I)$.
- $\tau_\Delta = 1.0$ 디폴트, $\tau_\Delta \to 0$ 이 v11t (symmetric t) 으로의 nested limit.
- **conditional-b kernel only** — v11tm history 에서 MIDUS 에 marginal_b 가 underperform
  했다는 결과 반영.

**Residual augmentation** 이 핵심: $r_q = b_q - \Delta_{c_q} U_q$ 가
v11t Gaussian-NIW conjugate 형태로 환원되어 weighted NIW machinery 그대로 재사용.

**새 Gibbs block 4개**:
- $U_q$: truncated $N_+(\mu_U, \sigma_U^2)$, $\sigma_U^{-2} = w_q(1 + \Delta^T \Sigma^{-1} \Delta)$
- $\Delta_l$: $N_d(M_l, V_l)$, $V_l^{-1} = (\sum w_q U_q^2)\Sigma^{-1} + \tau_\Delta^{-2} I$
- $w_q$: Gamma shape $(\nu_t+d+1)/2$, rate $(\nu_t + D_q + U_q^2)/2$
- $(\mu_l, \Sigma_l)$ NIW Gibbs on residuals (v11t 동일)

**c-update**: 잔차 t-predictive — $(n_{l,-q}+e_0) \cdot t_{\nu_n-d+1}(b_q - \Delta_l U_q; m_n, V_{pred})$

**Split-merge**: 신규 cluster (split) 는 parent Δ inherit; merge 는 surviving cluster
의 Δ 유지.  이후 sweep 의 Δ Gibbs 가 refresh.

### 2.7 Frühwirth-Schnatter & Pyne (2010) 와 비교

같은 SDB 형식의 latent representation 을 출발점으로 쓰지만, 거의 모든 디자인 선택이
정반대 방향:

| 측면 | FS-Pyne (2010) | v12_skewt |
|---|---|---|
| Inferential target | labeled component parameter | partition + biplot |
| K 결정 | post-hoc 모델비교 (BIC/AWE/ICL/DIC) | overfitted + Jain-Neal SM |
| $e_0$ 권장 | 큰 값 (K-stab) | 작은 값 (empty-prone) |
| $\nu$ | per-cluster MH | global fixed |
| Sampling block | joint $(\xi,\psi,\Sigma)$ + joint $(S,z)$ | sequential, residual augmentation |
| Identifiability | random permutation + relabeling | label-invariant summaries |
| Skew shrinkage | 없음 | $\tau_\Delta$ |
| Split-merge | 없음 | 있음 |

FS-Pyne 은 _density approximation 정확도_ 우선, v12_skewt 는 _partition inference + LSIRM
geometry_ 우선.  서로 다른 inferential 객체를 생산.

## 3. MIDUS driver 의 stale config 경고

`MIDUS_5layered_result_v12_skewt.R` 의 header 주석은 "v11t_stab lock-in
(kappa0=1.0, nu0=12, S0_init_scale=1)" 명시하지만 body 는 옛 loose 값
(kappa0=0.1, nu0=10, S0_init_scale=0.5) 로 남아있음.  v11tm clone 시 sync 안 함.
production 비교가 필요하면 313-316 줄을 lock-in 값으로 맞춰야 apples-to-apples.

또한 현재 driver 는 `nu_S0 = NA` 로 **S_0 고정** 상태 (Wishart hyperprior 비활성).
실험 의도에 맞게 `nu_S0 = d + 2` 로 켜는 것 고려.
